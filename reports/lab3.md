3. Было выяснено, каким образом можно настроить в командной строке приглашение ко вводу. В командной строке, чтобы изменить приглашение необходимо ввести следующее: `setx PROMPT "ЖЕЛАЕМОЕ ПРИВЕТСТВИЕ"`, команда `setx` предназначена для постоянного создания или изменения переменных среды в Windows. Установлено следующее приветствие: `setx PROMPT "$E[36m$T$H$H$H$H$H$H$E[0m $P$G"`. Было:
```
C:\Users\Тот Самый>
```
Стало (время без секунд + отображение синим цветом):
```
23:34 C:\Users\Тот Самый>
```
Список некоторых спецсимволов:
  - $P — Текущий диск и полный путь (самый важный код).
  - $G — Знак «больше» (>).
  - $L — Знак «меньше» (<).
  - $B — Символ вертикальной черты (конвейер — |).
  - $Q — Знак равенства (=).
  - $S — Пробел.
  - $T — Текущее время (часы, минуты, секунды, миллисекунды).
  - $D — Текущая дата.
  - $V — Номер версии Windows.
  - $N — Буква текущего диска.
  - $C — Левая скобка.
  - $F — Правая скобка.
  - $_ — Перевод строки (Enter). Позволяет сделать приглашение многострочным.
  - $E — Код Escape (ASCII-код 27). Используется для вставки цветов и управления курсором (ANSI-последовательности).
  - $H — Backspace (удаление предыдущего символа). Используется, например, чтобы обрезать миллисекунды в $T.
  - $$ — Сам знак доллара ($).

Также было изменено приглашение ко вводу в WSL-терминале. С помощью команды `nano ~/.bashrc` был открыт файл `.bashrc` в текстовом редакторе `nano`, где `.bashrc` — это конфигурационный файл командной оболочки (в нем хранятся настройки приглашения (PS1)). В конец файла было добавлено `export PS1='\[\e[36m\]\d \[\e[33m\]\t\[\e[0m\] \[\e[32m\]\u@\h\[\e[0m\]:\[\e[34m\]\w\[\e[33m\]$(__git_ps1 " (%s)")\[\e[0m\]\$ '` (Последовательно (кроме цветов): дата, время, имя пользователя, имя компьютера, ветка гит (если в репозитории), символ приглашения), эта строка переписала все предыдущие настройки приглашения. Выход из редактора `nano` был произведен с помощью горячих клавиш `Ctrl + X`, изменения сохранены. Было:

Стало:

В него была добавлена строка: `export PS1='\[\e[36m\]\d\[\e[0m\] \[\e[33m\]\t\[\e[0m\] \[\e[32m\]\u@\h\[\e[0m\] \w \[\e[96m\]Ветка гита:\[\e[0m\] \[\e[93m\]``__git_ps1 " (%s)"``\[\e[0m\]\n\$ '`. ### Эргономика работы с кодом

Чтобы посмотреть список горячих клавиш (+ назначить свои) в LibreOffice -- `Сервис -> Настройка -> Клавиатура`. Чтобы посмотреть список горячих клавиш VS Code -- `File -> Preferences -> Keyboard Shortcuts`. 

Интересные горячие клавиши VS Code: `Alt + Клик` (Поставить курсор в нескольких местах), `Ctrl + Shift + K` (удалить текущую строку), `Ctrl + Ё` (открыть встроенный терминал). 

  - поиск по содержанию файлов -- `Ctrl + Shift + F`;
  - перейти к объявлению функции или типа -- `Ctrl + G` (изначально этой клавиши не было, добавлена);
  - перейти к реализации функции или типа -- `Ctrl + Клик`;
  - найти все места, где используется переменная, функция или тип -- `Shift + F12`;
  - переименовать все вхождения переменной, функции или типа -- `F2`;
  - быстро перемещаться между несколькими наиболее интересными местами кода -- `Alt + <-` / `Alt + ->`;
  - комментировать и раскомментировать большие части кода -- `Ctrl + /`.

### Кастомизация процессов

В VS Code для создания своей горячей клавиши нужно открыть keybindings.json и добавить строку вида { "key": "ctrl+shift+b", "command": "workbench.action.tasks.build" }. Открыть этот файл можно через `File -> Preferences -> Keyboard Shortcuts -> Значок файла`. Можно присвоить действиям предусмотренным в VS code горячие клавиши, или же создать свое действие и присвоить ему горячую клавишу. Свои действия можно создавать через User Tasks. Так были добавлены следующие горячие клавиши:
  - `Ctrl + Alt + D` -- make debug в текущей директории
  - `Ctrl + Alt + L` -- make leaks в текущей директории 
  - `Ctrl + Alt + T` -- make tests в текущей директории
  - `Ctrl + Alt + C` -- make clean в текущей директории

### Интеграция с гитом

Для интеграции с гитом многого сделано не было: нажата иконка с веточками слева в VS Code. Далее кнопки согласия, теперь слева в панели отображается Changes (отображает где были изменения, а также предлагает сделать коммит) и Graph (дерево веток и лепестки коммитов). 

  - просмотр изменений -- Нажать на интересуемый файл в разделе Changes,
  - добавление файлов в индекс -- ,
  - коммиты,
  - добавление удаленного репозитория,
  - получение обновлений из удаленного репозитория,
  - отправка обновлений в удаленный репозиторий,
  - разрешение конфликтов.
