# Лабораторная работа №3. Настройка среды разработки

## Цель работы

Познакомиться с базовыми приемами настройки инструментов
разработки, с помощью которых можно удобнее настроить рабочие процессы.

## Этапы выполнения работы

### Для начала

1. Было зафиксировано следующее (действие -- в какой программе):

  - редактирование текстовых файлов -- Блокнот, LibreOffice, VS Code;
  - программирование -- VS Code;
  - для выполнения команд в консоли -- Терминал;
  - для навигации по файловой системе -- Windows Проводник;
  - для работы с гитом -- Терминал, Git Bash.

2. Альтернативы для программ: 
  
  - редактирование текстовых файлов -- Notepad++ (поддерживает подсветку синтаксиса для 50+ языков, работу с вкладками), Obsidian (создание базы знаний, работает с Markdown);
  - программирование -- Sublime Text (быстрый редактор, подходит для верстки и быстрого редактирования), Cursor (редактор на базе VS Code с интеграцией AI-помощников), CLion (среда разработки для языков C и C++);
  - для выполнения команд в консоли -- Cmder (портативный эмулятор консоли для Windows, созданный для замены стандартной командной строки более функциональным и визуально приятным интерфейсом), Hyper (современный эмулятор терминала с открытым исходным кодом);
  - для навигации по файловой системе -- Total Commander (двухпанельный интерфейс, мощный инструмент группового переименования, поддержка работы с архивами как с папками), Files (поддержка вкладок, двухпанельный режим, кастомизация интерфейса и теги для файлов);
  - для работы с гитом -- VS Code, Cmder.

  В силу долгой привычки, альтернативы программ для программирования, выполнения команд в консоли, навигации по файловой системе, а также для работы с Git не произвели впечатления. Однако опробованный редактор текстовых файлов [Obsidian](https://obsidian.md/) проявил себя как нечто особенное и ранее неведомое. Это мощная альтернатива для работы с текстом, особенно для заметок и личных записей. В отличие от Блокнота, Obsidian умеет связывать заметки между собой, работать с Markdown, имеет кучу (если не тучу) плагинов. Заменой Блокнота и VS Code он не стал, но остался для личных заметок на будущее.
